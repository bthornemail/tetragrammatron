# Track D Charter — Productization

## Objective

Improve delivery ergonomics and operator usability without introducing semantic behavior.

## In

- Browser/static Hub delivery
- Installer/dev ergonomics
- Packaging/binary distribution
- Docs and onboarding polish

## Out

- Protocol semantics
- Capability verification rules
- Federation arbitration logic

## Acceptance gates

- Install/run path is reproducible from clean state
- Hub delivery remains projection-only
- Packaging does not alter canonical output behavior

## Milestones

- D1: delivery target selection + packaging scripts
- D2: hub static/browser integration and docs
- D3: release channel and onboarding hardening
